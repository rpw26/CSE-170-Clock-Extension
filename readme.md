# CSE 170 Clock Extension

Because many people will be working on this, we have set up a system so that there are little to no merging issues.

## Guide on how to develop

this is explained [here](./guide.md).

## How to contribute

1. Make sure you have a **fork** of this repository like so:<br>

   <img width="516" src="https://github.com/user-attachments/assets/3fc65d22-05ad-434c-8d20-74b24d782800" />

2. Then copy the link and add it to your **local GitHub Desktop** (or you can use the command line):<br>

   <img width="516" src="https://github.com/user-attachments/assets/1735059f-d27e-4c0c-ac8b-c1387ce49ddb" />

3. Clone the repository to your computer using **GitHub Desktop**.

4. Before starting work, make sure your fork is up to date:
   - Go to your fork on GitHub.
   - Click **Sync fork → Update branch**.
   - Then **pull the latest changes** in GitHub Desktop.

5. Make your changes to the project files.

6. In **GitHub Desktop**, commit your changes with a short message describing what you did.  
   Example commit messages:
   - `added seconds display`
   - `fixed alignment issue`
   - `improved clock styling`

7. Click **Push origin** to upload your changes to your fork.

8. Go to your fork on **GitHub.com**. You should see a button that says **Compare & Pull Request**.

9. Click it and create a **Pull Request** to this repository.

10. A repository owner will review the changes and either:
   - **Merge the request** (accept it)
   - **Request changes**
   - **Close the request** (decline it)

## Workflow Summary

Fork → Clone → Sync Fork → Pull → Make Changes → Commit → Push → Pull Request → Review → Merge

## IMPORTANT

You only need to create a fork **once** when setting up. After that, you can work like a normal repository.

Always make sure your fork is up to date before programming. This helps prevent accidentally overwriting someone else's work.

If your code overwrites someone else's work or causes conflicts, it will likely not be accepted.
